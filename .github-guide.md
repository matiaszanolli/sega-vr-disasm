# GitHub Publishing Guide

Quick reference for publishing this repository to GitHub.

## Initial Setup

### 1. Create GitHub Repository

Go to GitHub and create a new repository:
- **Name**: `virtua-racing-deluxe-disassembly` (or your preference)
- **Description**: "Complete, buildable disassembly of Virtua Racing Deluxe (32X) achieving perfect byte-for-byte ROM match"
- **Public/Private**: Your choice
- **DO NOT** initialize with README (we have one)

### 2. Initialize Local Git

```bash
cd /mnt/data/src/32x-playground

# Initialize git if not already done
git init

# Add all files (respects .gitignore)
git add .

# Check what will be committed
git status

# Should show:
# - All .md files
# - Makefile, .gitignore, LICENSE
# - disasm/*.asm files
# - tools/*.py files
# - docs/*.md files
# - analysis/*.md files
# Should NOT show:
# - *.32x files
# - *.bin files
# - build/*.32x files
# - tools/vasmm68k_mot
```

### 3. Initial Commit

```bash
git commit -m "Initial commit: Perfect byte-for-byte ROM match achieved

- Complete disassembly of ROM header (512 bytes)
- Custom M68K and SH2 disassemblers
- Full build system with vasm
- Comprehensive 32X hardware documentation
- Perfect MD5 match: 72b1ad0f949f68da7d0a6339ecd51a3f

Ready for incremental code disassembly."
```

### 4. Push to GitHub

```bash
# Add remote (replace with your repository URL)
git remote add origin https://github.com/yourusername/virtua-racing-deluxe-disassembly.git

# Push to main branch
git branch -M main
git push -u origin main
```

## Creating a Release

When ready to tag a release:

```bash
# Tag the current state
git tag -a v1.0 -m "Version 1.0: Perfect ROM match achieved

- Complete ROM header disassembly
- Build system fully functional
- All documentation complete
- MD5: 72b1ad0f949f68da7d0a6339ecd51a3f"

# Push the tag
git push origin v1.0
```

Then on GitHub:
1. Go to Releases
2. Click "Draft a new release"
3. Select tag `v1.0`
4. Title: "v1.0 - Perfect ROM Match Achieved"
5. Description: (copy from PROGRESS.md)
6. Attach: (none needed - everything is in the repo)
7. Publish release

## GitHub Actions (Optional CI/CD)

Create `.github/workflows/build.yml`:

```yaml
name: Build ROM

on: [push, pull_request]

jobs:
  build:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v2

      - name: Install dependencies
        run: sudo apt-get install -y build-essential python3 wget

      - name: Build vasm
        run: make tools

      # Note: Can't test full build without ROM
      # But can test tools
      - name: Test disassemblers
        run: |
          python3 tools/m68k_disasm.py --help || true
          python3 tools/sh2_disasm.py --help || true
```
